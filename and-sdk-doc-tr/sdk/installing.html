This page describes how to install the latest version of the Android SDK and set up your development environment for the first time.

这一页介绍了如何安装最新版本的Android SDK，并搭建开发环境。

If you encounter any problems during installation, see the Troubleshooting section at the bottom of this page.

如果您在安装过程中遇到问题请参看页底部的Troubleshooting一 节尝试解决。

Updating?——升级？

If you are currently using the Android 1.6 SDK, you do not necessarily need to install a newer SDK, since you can already update the platforms, tools, and other components using the Android SDK and AVD Manager tool. To develop against the latest Android platform, for example, you could just download the latest SDK Tools and then add the new Android platform into your existing SDK.

如果您目前使用Android1.6 SDK，你不一定需要安装一个新的SDK的，因为你已经可以使用Android SDK and AVD Manage更新平台、工具和其他开发组件。为了在最新的Android平台中开发，例如，您可以只下载最新的SDK工具，然后添加新 的Android平台到您现有的SDK中。

If you are using Android 1.5 SDK or older, you should install the new SDK as described in this document and move your application projects to the new SDK environment.

如果您正在使用Android 1.5 的SDK或更老的版本，您则必须安装这篇文档中所述的新版的SDK然后将您的软件工程转移到新的SDK环境中。

Prepare for Installation——准备安装

Before you begin, take a moment to confirm that your development machine meets the System Requirements.

在开始之前，请花点时间来确认您的开发用计算机满足 System Requirements 。

If you will be developing in Eclipse with the Android Development Tools (ADT) Plugin — the recommended path if you are new to Android — make sure that you have a suitable version of Eclipse installed on your computer (3.4 or newer is recommended). If you need to install Eclipse, you can download it from this location:

如果您决定使用Eclipse及ADT插件来进行Android开发——建议您这样，如果您是新手— —确认您计算机上安装的是符合要求的Eclipse版本（建议3.4或更新版本）。如果您需要安装Eclipse，您可从此下载：

http://www.eclipse.org/downloads/

A Java or RCP version of Eclipse is recommended. For Eclipse 3.5, the "Eclipse Classic" version is recommended.

推荐您使用JAVA或RPC版本的Eclipse。对Eclipse 3.5版，推荐使用“Eclipse Classic”版。

Download and Install the SDK——下载并安装SDK

Download the SDK package that is appropriate for your development computer. You can get the latest version from the SDK download page.

下载针对您的开发用计算机系统环境的SDK包。您可从SDK download page获得最新版SDK。

After downloading, unpack the Android SDK archive to a suitable location on your machine. By default, the SDK files are unpacked into a directory named android-sdk-<machine-platform>. Make a note of the name and location of the unpacked SDK directory on your system — you will need to refer to the SDK directory later, when setting up the ADT plugin or when using the SDK tools.

下载后，在您的计算机上将压缩包解压到合适位置。默认的，压缩包被解压到名为android-sdk-<machine-platform>的 文件夹。记住在您系统上解压后的SDK所在文件夹路径及名称——您需要在过后引用此路径，即在设置ADT插件和使用SDK工具 时。

Optionally, you may want to add the location of the SDK's primary tools directory to your system PATH. The primary tools/ directory is located at the root of the SDK folder. Adding tools to your path lets you run Android Debug Bridge (adb) and the other command line tools without needing to supply the full path to the tools directory. 

可选的，您可能会想将SDK的主要工具的文件夹路径加入您的系统环境变量中。主要的tools/文件夹就在SDK文件夹 下。将tools加入您的系统环境变量中后，您可以从命令行中直接运行Android 调试连接桥（adb），而不必使用其完 整路径。

  • On Linux, edit your ~/.bash_profile or ~/.bashrc file. Look for a line that sets the PATH environment variable and add the full path to the tools/ directory to it. If you don't see a line setting the path, you can add one:在Linux上，编辑您的~/.bash_profile或~/.bashrc文 件。寻找设置PATH环境变量的一行，然后将tools/的完整路径添加于其后。如果您未找到设置 路径的行，您可直接添加一行。
      • export PATH=${PATH}:<your_sdk_dir>/tools
  • On a Mac OS X, look in your home directory for .bash_profile and proceed as for Linux. You can create the .bash_profile if you haven't already set one up on your machine.在Mac OS X上，在您的主文件夹下寻找.bash_profile文 件，然后像在Linux中一样修改它。如果未找到，您可新建一个.bash_profile文件。
  • On Windows, right-click on My Computer, and select Properties. Under the Advanced tab, hit the Environment Variables button, and in the dialog that comes up, double-click on Path (under System Variables). Add the full path to the tools/ directory to the path.在Windows中，在我的电脑上单击右键，选择属性。在高级选项卡中，点击环境变量按钮，在弹出的对话框中双击Path（在系统变量下边）。 添加到 tools/文件夹的完整路径。

If you will be using the Eclipse IDE as your development environment, the next section describes how to install the Android Development Tools (ADT) plugin and set up Eclipse. If you choose not to use Eclipse, you can develop Android applications in an IDE of your choice and then compile, debug and deploy using the tools included in the SDK (skip to Next Steps).

如果您使用Eclipse IDE作为您的开发环境，您应该继续下面的章节，它们讲述了如何安装Android开发工具插件（ADT）和如何设置 Eclipse。如果您选择不使用Eclipse，您可以在您习惯的IDE中开发，然后使用SDK中提供的工具进行编译，调试，部署。并且您可以跳过下面 的章节，去到Next Steps。

Install the ADT Plugin for Eclipse——安装Eclipse的ADT插件

Android offers a custom plugin for the Eclipse IDE, called Android Development Tools (ADT), that is designed to give you a powerful, integrated environment in which to build Android applications. It extends the capabilites of Eclipse to let you quickly set up new Android projects, create an application UI, add components based on the Android Framework API, debug your applications using the Android SDK tools, and even export signed (or unsigned) APKs in order to distribute your application.

Android 提供了一个定制的Eclipse IDE 的插件，叫做Android开发工具（ADT），旨在为您提供一个强大的，集成的环境来 建立Android应用程序。它扩展了Eclipse的功能来使您快速地建立新Android工程，建立程序UI，添加基于 Android框架API（Android Framework API） 的组件，使用Android工具调试您的应用程序，甚至导出已签名的（或未签名的）程序安装包来进行应用程序的发布。

In general, using Eclipse with ADT is a highly recommended approach to Android development and is the fastest way to get started. For complete information about how to install ADT, see Installing and Updating ADT.

总的来说，强烈建议使用带ADT插件的Eclipse进行Android开发，而且，它是开始Android开发的最快速的方法。如何安装ADT的 详细方法，请参见Installing and Updating ADT 。

If you prefer to work in an IDE other than Eclipse, you do not need to install Eclipse or ADT, instead, you can directly use the SDK tools to build and debug your application.

如过您更习惯其他的IDE，您就不需要安装Eclipse和ADT，然后，您可以直接使用SDK工具建立并调试您的应用程序。

Add Android Platforms and Other Components——添加Android平台和其他组件

The Android SDK and AVD Manager tool is pre-installed in your SDK. Using the tool is a key part of performing the initial setup of your SDK, as well as keeping it up-to-date with the latest platforms, tools, and other components.

Android SDK及AVD管理工具（Android SDK and AVD Manager）已经预装 在您下载的SDK包中。您必须使用此工具进行SDK的初始安装，您也必须使用它来更新平台，工具以及其他组件。

For full instructions on how to use the tool, see Adding SDK Components.

使用此工具的完整介绍，请参见 Adding SDK Components。

The Android SDK uses a modular structure that separates the major parts of the SDK — platforms, add-ons, tools, and the API documentation — into a set of separately installable components. The SDK components are available to you for individual download, as needed, from the Android SDK repository site.

Android的SDK的采用模块化结构，分离的SDK中的主要部分  ——平台，附加部分，工具和API文档——成一个个单独安装的组件集 。SDK组件可供您根据需要从Android SDK的存放点分开下载。

The Android SDK starter package includes only a single component: the latest version of the SDK Tools. Included in that component is a tool called Android SDK and AVD Manager that you can use to download other components from the SDK repository site. The tool provides a graphical UI that lets you browse the repository, select new or updated components for download, and then install them in your SDK.

Android包初始时只有一个组件：最新版的SDK工具。其中包含了一个用以从SDK存放站点下载其他的SDK组件工具— —Android SDK and AVD Manager。该工具提供了一个图形界面，使您能浏览SDK存放站点，选择新的组件进行下载或更新，然后安装在您的 SDK中。

There are several types of SDK components available:

SDK组件有以下几种：

  • SDK Tools (available pre-installed in the Android SDK package) — Contains the full set of SDK tools for developing, debugging, and testing your application code and UI. You can read about the tools in the Dev Guide and access them in the <sdk>/tools/ directory.     SDK 工具（已经预装在Android SDK包中，并可直接使用）——包含了全部的开发、调试和测试您应用程序代码和UI的工具。您可从Dev Guide获得它们的信息，在<sdk>/tools/文 件夹下找到它们。
  • Android platforms — An SDK platform component is available for every production Android platform deployable to Android-powered devices. Each platform component includes a fully compliant Android library and system image, sample code, emulator skins, and any version specific tools. For detailed information about each platform, see the overview documents available under the section "Downloadable SDK Components," at left.     Android 平台——SDK平台组件适用于每一个可安装Android或特为 Android设计的设备产品。每个平台组件包含了一个完全兼容的Android库和系统镜像、示例代码、模拟器皮肤外观以及所有版本特别的工具。每个版 本的平台的详细信息请参见左侧“Downloadable SDK Components——可供下载的SDK组件” 章节下边的概述文档。
  • SDK Add-Ons — SDK add-ons provide a development environment for specific Android external library or a customized (but fully compliant) Android system image. The Android SDK repository offers the Google APIs Add-On, which gives your application access to powerful mapping capabilities through the com.google.android.maps library. You can also add additional repositories, so that you can download other SDK add-ons, where available.     SDK 附加部分——SDK附加部分 提供了特定扩展库或定制的（但是是完全兼容的）Android系统镜像。Android SDK存放站点提供了Google APIs附加部分，它能使你的应用程序通过com.google.android.maps库获得强大的地图功能。您也可以添 加其他的存放站点，从上面下载其他SDK附加部分。
  • Documentation — Contains a local copy of the latest multiversion documentation for the Android framework API.      文档——包含了多个版本的Android框架API文档的本地拷贝。

To develop any Android application, even if you are following the Hello World tutorial, you must download at least one Android platform into your SDK. Typically, you will want to download multiple platforms, so that you can build your application on the lowest version you want to support, but test against higher versions that you intend the application to run on. You can test your applications on different platforms by running in an Android Virtual Device (AVD) on the Android emulator.

开发任何Android应用程序，哪怕只是照着Hello World入门指南一步一步做，您都必须下载至少一个版本的 Android平台到您的SDK中。一般来说，您会下载多个平台，在您想支持的最低 版本上编写应用程序，然后在您期望它实际运行的更高版本上测试。您可以使用Android模拟器运行不同版本平台的Android虚拟设备（AVD）进行 测试。

For step-by-step instructions on how to use the Android SDK and AVD Manager to add components, see the Adding SDK Components document.

一步步介绍如何使用Android SDK和AVD管理工具来添加组件，请参见Adding SDK Components文档。

For release notes and other detailed information about individual SDK components, see the documents listed under "Downloadable SDK Components" in the navigation at left.

获得单个SDK组件的版本说明和详细信息请参见左侧导航控制板中列在"Downloadable SDK Components——可下载的SDK组件"下边的文档。

Explore the SDK——SDK文件夹结构

Once you've installed the SDK and downloaded the platforms, documentation, and add-ons that you need, open the SDK directory and take a look at what's inside.

在安装好SDK并下载了您需要的平台、文档和附加部分后，让我们打开SDK文件夹，看看里面都包含些什么。

The table below describes the full SDK directory contents, with components installed.

下边表格列出了完整的SDK文件夹的内容，包括安装的组件。

Name Description
add-ons/ Contains add-ons to the Android SDK development environment, which let you develop against external libraries that are available on some devices.
包含了Android SDK的附加部分，能让您使用有些设备上提供的扩展库。
docs/ A full set of documentation in HTML format, including the Developer's Guide, API Reference, and other information. To read the documentation, load the file offline.html in a web browser.
包含HTML格式的完整的文档，包括开发指导、API参考索引和其他信息。使用网页浏览器加载offline.html进行阅读。
platforms/ Contains a set of Android platform versions that you can develop applications against, each in a separate directory.
包含了一系列版本的可用于开发的Android平台，每一个存放于单独的文件夹。

<platform>/ Platform version directory, for example "Android 1.6". All platform version directories contain a similar set of files and subdirectory structure.
以平台版本命名的文件夹，例如：“Android 1.6”。所有平台的文件夹的文件和结构基本相同。
 
data/ Storage area for default fonts and resource definitions.
默认字体和资源定义的存放区


images/ Storage area for default disk images, including the Android system image, the default userdata image, the default ramdisk image, and more. The images are used in emulator sessions.
默认磁盘镜像的存储区，包括Android系统镜像、默认用户数据的镜像、默认的内存磁盘的镜像等等。这些镜像用于模拟器的模拟。


samples/ Contains a wide variety of sample applications that you can load as projects into your development environment, compile, and run on the emulator.
包含了多种多样的示例程序，您可将他们作为项目导入开发环境中，编译，然后在模拟器中运行。


skins/ A set of emulator skins available for the platform version. Each skin is designed for a specific screen resolution.
包含了这种平台可用的模拟器外观集合。每一种外观使用了一种屏幕分辨率。


templates/ Storage area for file templates used by the SDK development tools.
SDK开发工具所使用的文件模板的存放区。


tools/ Any development tools that are specific to the platform version.
所有此种平台特有的开发工具。


android.jar The Android library used when compiling applications against this platform version.
使用这个版本的平台编译应用程序时使用的Android库。
tools/ Contains the set of development and profiling tools available to you, such as the emulator, the android tool, adb, ddms, and more.
包含了一系列的开发和分析工具，例如模拟器、android 工具、adb、ddms等等。
SDK Readme.txt A file that explains how to perform the initial setup of your SDK, including how to launch the Android SDK and AVD Manager tool on all platforms
介绍如何进行SDK初始安装的帮助文件，包括如何在所有的系统平台上运行Android SDK 和 AVD管理工具。
SDK Setup.exe For Windows SDK only. A shortcut that launches the Android SDK and AVD Manager tool, which you use to add components to your SDK.
只有Windows上SDK有此文件。加载Android SDK和AVD管理工具的快捷方式，Android SDK和AVD管理工具被用来添加SDK组件。

Next Steps——下一步

Once you have completed installation, you are ready to begin developing applications. Here are a few ways you can get started:

完成安装之后，您就可以开始应用程序的开发了，下面是开始您Android编程之旅的几种方法：

Learn about Android——学习Android

  • Take a look at the Dev Guide and the types of information it provides     参看Dev Guide和它所提供的信息
  • Read an introduction to Android as a platform in What is Android?     从What is Android中阅读Android平台的介绍
  • Learn about the Android framework and how applications run on it in Application Fundamentals     从Application Fundamentals中了解Android的架构和应用程序的运行机制
  • Take a look at the Android framework API specification in the Reference tab     从Reference标签下查看Android框架API规范

Explore the SDK——SDK文件夹结构

  • Get an overview of the development tools that are available to you     总览一下您可以使用的development tools（开发工具）
  • Read how to develop in Eclipse/ADT or in other IDEs     阅读有关如何在 Eclipse/ADT或other IDEs中进行开发

Explore some code——浏览一些代码

  • Set up a Hello World application (highly recommended, especially for Eclipse users)     建立一个Hello World application（强烈推荐，尤其是使用Eclipse进行开发的用户）
  • Follow the Notepad Tutorial to build a full Android application     跟随 Notepad Tutorial的介绍建立一个完整的Android应用程序
  • Create a new project for one of the other sample applications included in <sdk>/platforms/<platform>/samples, then compile and run it in your development environment     为<sdk>/platforms/<platform>/samples中的另外一个示例程序建立新的项目，然后在您的开发环境中编译，运行

Visit the Android developer groups——访问Android开发者群组

  • Take a look at the Community tab to see a list of Android developers groups. In particular, you might want to look at the Android Developers group to get a sense for what the Android developer community is like.     点击Community标签查看Android开发者群组的列表。特别的，您可能会希望通过Android Developers群组来感受Android开发者社区是什么样的。

Troubleshooting——常见问题及解答

Ubuntu Linux Notes——Ubuntu Linux 注意事项

  • If you need help installing and configuring Java on your development machine, you might find these resources helpful:     如果您希望获得有关如何在您的开发用计算机上安装和配置Java，请参考以下文章
    • https://help.ubuntu.com/community/Java
    • https://help.ubuntu.com/community/JavaInstallation
  • Here are the steps to install Java and Eclipse, prior to installing the Android SDK and ADT Plugin.     这里是安装Java 和Eclipse的步骤，然后您才能安装Android SDK和ADT插件
    1. If you are running a 64-bit distribution on your development machine, you need to install the ia32-libs package using apt-get::     如果在您开发用计算机上运行的是一个64位的发行版本，您需要使用apt-get来安装ia32-libs包： 
      apt-get install ia32-libs
    2. Next, install Java:     然后，安装Java 
      apt-get install sun-java6-bin
    3. The Ubuntu package manager does not currently offer an Eclipse 3.3 version for download, so we recommend that you download Eclipse from eclipse.org (http://www.eclipse.org/ downloads/). A Java or RCP version of Eclipse is recommended.     Ubuntu包管理程序现在还未提供Eclipse 3.3版本的下载，因此我们建议您从eclipse.org (http://www.eclipse.org/ downloads/)下载Eclipse。建议使用Java或RCP版本的Eclipse。
    4. Follow the steps given in previous sections to install the SDK and the ADT plugin.     按照前面章节给出的步骤来安装SDK和ADT插件。

Other Linux Notes——其他Linux的注意事项

  • If JDK is already installed on your development computer, please take a moment to make sure that it meets the version requirements listed in the System Requirements. In particular, note that some Linux distributions may include JDK 1.4 or Gnu Compiler for Java, both of which are not supported for Android development.     如果您的开发用计算机上已经安装了JDK，请花一些时间，以确保它符合在系统要求中列出的版本要求。特别是，注意，一些Linux发行版本可能包括JDK 1.4或Java的GNU编译器，这两者都是不支持Android开发的。
